# PSD → 预制体 图层命名规范

> 适用于 Cocos Creator 2.4.15 的 PSD 转预制体工具。
> 转换工具按本规范解析每个 PSD 图层 / 图层组的名字,决定**节点类型、是否创建、是否复用图片、组件参数**。
> 命名格式统一为:`[前缀_]名字[ #后缀][ @修饰]`,大小写不敏感,分隔符用下划线 `_`。

---

## 一、节点类型前缀

工具根据前缀决定生成什么节点 / 挂什么组件。**普通位图图层不写前缀时默认生成 `cc.Sprite` 节点**。

| 前缀 | 生成节点 | 说明 |
|------|----------|------|
| (无) | Node + cc.Sprite | 默认。普通图片层 |
| `lbl_` | Node + cc.Label | 文本层,保留文字内容、字号、颜色 |
| `rt_` | Node + cc.RichText | 富文本 |
| `btn_` | Node + cc.Button + cc.Sprite | 按钮,自动找子层做各状态(见第四节) |
| `tog_` | Node + cc.Toggle | 开关 |
| `sp_` | Node + cc.Sprite (SLICED) | 九宫格图,需配 `#` 后缀指定边距 |
| `prog_` | Node + cc.ProgressBar | 进度条 |
| `node_` | 空 Node(无组件) | 纯容器 / 占位锚点 |
| `mask_` | Node + cc.Mask | 遮罩容器 |
| `sv_` | Node + cc.ScrollView | 滚动视图 |
| `edit_` | Node + cc.EditBox | 输入框 |
| `lay_` | Node + cc.Layout | 自动布局容器(见第三节) |

---

## 二、图层组(容器)规则

图层组默认转成**空 Node 容器**,子层成为其子节点。

- 组名不带前缀 → 普通容器 Node。
- 组名带上表前缀(如 `btn_`、`lay_`、`sv_`)→ 容器挂对应组件。
- 容器节点的尺寸 = 子层包围盒的并集(可配置成不计算尺寸)。
- 容器锚点统一 `(0.5, 0.5)`,位置按其内容包围盒中心换算。

---

## 三、布局容器 `lay_`

`lay_` 容器自动挂 `cc.Layout`,布局方向用修饰段指定:

| 命名 | Layout Type |
|------|-------------|
| `lay_h_列表` | HORIZONTAL |
| `lay_v_列表` | VERTICAL |
| `lay_grid_列表` | GRID |

间距等参数转换后在编辑器里微调,或扩展里设默认值。

---

## 四、按钮 `btn_` 的状态子层

`btn_xxx` 组内部用固定子层名提供各状态图,工具自动连到 Button 的 transition:

| 子层名 | 对应 |
|--------|------|
| `normal` | 普通态(必需,作为 Button 的 target Sprite) |
| `pressed` | 按下态 |
| `hover` | 悬停态 |
| `disabled` | 禁用态 |
| 其它子层(如 `lbl_文字`、图标) | 作为按钮的普通子节点正常生成 |

若只有一张图、无状态子层 → 生成 SCALE 过渡的按钮。

---

## 五、九宫格 `#` 后缀

`#` 后缀声明九宫格边距(SLICED),单位像素,顺序 `上,右,下,左`(同 CSS):

| 命名 | 含义 |
|------|------|
| `sp_框 #20` | 四边都留 20px |
| `sp_框 #20,30` | 上下 20,左右 30 |
| `sp_框 #10,20,10,20` | 上 10 右 20 下 10 左 20 |

不写 `sp_` 前缀但带 `#` 的,也按九宫格处理。

---

## 六、修饰后缀 `@`

可叠加,空格分隔,放在名字最后。

| 修饰 | 作用 |
|------|------|
| `@2x` / `@3x` | 倍图。导出后节点尺寸 = 像素 / 倍数,实现高清图小尺寸显示 |
| `@nocrop` | 不裁剪透明边,按图层名义包围盒导出(需保持留白对齐时用) |
| `@flipx` / `@flipy` | 翻转,复用同一张图、只翻 scale,常配合去重 |
| `@gray` | 置灰(挂置灰材质或导出灰度图) |
| `@90` `@180` `@270` | 旋转复用同一张图 |

---

## 七、忽略 / 不创建节点

命中以下任一条件的图层**不生成节点、不导出图片**:

| 标记 | 说明 |
|------|------|
| 名字以 `//` 开头 | 注释层,纯参考 |
| 名字以 `!` 开头 | 显式忽略 |
| `ref_` 前缀 | 美术参考图(参考线、对齐底图) |
| `tmp_` 前缀 | 临时层 |
| `bg_guide` 等约定名 | 自定义辅助层 |
| 图层不可见(隐藏) | 默认跳过(可配置保留并设 active=false) |
| 不透明度为 0 | 跳过 |
| 裁剪后宽或高为 0 | 空层,跳过 |
| 空图层组(子层全被过滤) | 递归跳过 |

> 想保留隐藏层但默认关闭,用 `keep_` 前缀:生成节点但 `active=false`。

---

## 八、图片去重(MD5)相关

去重由工具自动做,命名上只需注意:

- 想**强制独立**一张图(不参与去重复用),加 `@unique` 修饰。
- 想**强制指向已有资源**,用 `=资源名` 显式引用,如 `icon=common_star`,工具不导出新图,直接用项目里 `common_star` 的 spriteFrame。
- 翻转 / 旋转复用统一走 `@flipx` `@90` 等修饰,避免导出镜像重复图。

## 九、命名示例

```
PSD 图层树                              生成结果
─────────────────────────────────────────────────────────
弹窗                                    Node「弹窗」(容器)
├─ sp_背景 #40                          Sprite, 九宫格边距40
├─ // 设计参考线                         (忽略,不创建)
├─ lay_v_列表                            Node + Layout(垂直)
│   ├─ item                              Node「item」
│   │   ├─ icon=common_star             Sprite, 复用已有 common_star
│   │   └─ lbl_数量                      Label「数量」
│   └─ item_副本                         (内容MD5相同→图片复用)
├─ btn_确定                              Node + Button
│   ├─ normal                           普通态图
│   ├─ pressed                          按下态图
│   └─ lbl_文字                          Label 子节点
├─ icon_装饰 @2x                         Sprite, 2倍图缩半显示
└─ ref_对齐底图                          (忽略)
```

## 十、约定优先级(冲突时)

1. 忽略规则(第七节)最高 —— 命中即跳过,其它一概不处理。
2. 显式引用 `=资源名`(第八节)—— 跳过导出,直接连资源。
3. 类型前缀(第一节)决定组件。
4. `#` / `@` 修饰叠加在上面调参数。

## 十一、输出的节点命名规则(工具自动处理)

生成预制体时,节点名按统一规则产生,无需在 PSD 里手动遵守:

1. **全英文**:已有英文保留,中文转拼音首字母大写;Label 显示文本保留原文。
2. **类型前缀**:图片 `Img_`、文字 `Label_`、无图无文字的容器/空节点 `Node_`
   (按钮、布局等容器都归 `Node_`);预制体根节点保留干净名、不加前缀。
3. **后缀长度 ≤ 8**:前缀之后的部分最多 8 个字母/数字,超出截掉(下划线不计);
   **截断时优先保留末尾数字**(`形状590`→`XingZ590`),减少重名。
4. **同父节点下名字唯一**:重名自动加 `_2/_3`,保证 `getChildByName`/`cc.find` 能定位。
5. **压平冗余空容器**(默认开):无组件、只含 1 个子的空组会被合并,并保留外层有意义的名字。
6. **合并重叠装饰图**(可选):同位置、尺寸相近的相邻兄弟图只留一张(改变外观,默认关)。

## 十二、给"写代码"的命名建议(美术可逐步采用)

工具能保证结构和唯一性,但**给不了语义**——PS 默认名(形状/矩形/图层/组)和图片哈希名
对程序员没用。要让预制体好写代码,建议美术对**代码会用到的层**做到:

- **起有意义的名字**:`btn_start`、`lbl_coin`、`icon_vip`,而不是留 `形状590`。无关装饰层可随意。
- **该交互的用 `btn_`**:底部 tab、确定/取消等,工具会自动生成 `cc.Button`。
- **列表/排列用 `lay_h_`/`lay_v_`/`lay_grid_`**:工具自动挂 `cc.Layout`。
- **重复结构**(如多个同款格子)建议在 PSD 里只做一个、命名清晰,代码侧用一个子预制体 + 数据驱动。

这些都是**渐进兼容**的:美术现在不改也能转;改了之后工具会自动识别成对应组件,无需改工具。

---

### 落地建议
- 工具里把「忽略前缀表、类型前缀表、去重表路径、是否保留隐藏层」做成配置项,方便不同项目改规则。
- 团队约定:**所有要进预制体的层必须显式命名**,临时层一律加 `tmp_` / `//`,避免脏层混入。
